# ComindFlow 技能系统重构 - 总览

**版本**: 1.0
**日期**: 2026-06-29
**状态**: 草案

---

## 背景 (Background)

ComindFlow 项目当前拥有 10+ 个 Skills（grill-me、grill-you、advance-task 等），采用文件接口适配模式与 Agent Framework 集成。随着技能数量增长和复杂度提升，现有架构面临以下挑战：

### 当前痛点

| 问题 | 影响 |
|------|------|
| **硬编码依赖** | Workflow 节点直接引用具体 Adapter 类，扩展性差 |
| **全量加载** | 所有 SKILL.md 内容完整加载到上下文，Token 消耗高 |
| **缺乏可观测性** | Skill 执行过程无链路追踪，调试困难 |
| **标准不一** | SKILL.md 格式差异大，复用率低 |

### 重构目标

构建基于 **动态按需注入（Dynamic On-demand Injection）** 范式的高扩展性 Agent 技能系统。

---

## 重构范围 (Scope)

### IN SCOPE（包含）

| 组件 | 说明 |
|------|------|
| **Skill Registry** | 自动发现、注册、管理 Skills |
| **Middleware 调度层** | 请求拦截、上下文管理、技能路由 |
| **SKILL.md 标准化** | 遵循 write-a-skill 规范 |
| **上下文优化** | 目录注入 + 按需加载 |
| **可观测性** | 链路追踪、异常诊断、幻觉检测 |

### OUT OF SCOPE（不包含）

| 项目 | 说明 |
|------|------|
| **Skill 行为变更** | 保持现有 Skills 的业务逻辑不变 |
| **LLM 集成方式** | 仍通过 Claude Code CLI 调用 |
| **多租户支持** | 单用户本地运行 |

---

## 核心需求 (Core Requirements)

### 需求 1：技能注册表与中间件调度层

**目标**：实现业务技能与模型逻辑的解耦

| 子需求 | 说明 |
|--------|------|
| Skill Registry | 扫描 SKILL.md 文件，自动注册技能 |
| Middleware 调度层 | 请求拦截、上下文管理、技能路由 |
| 动态选择 | 运行时根据任务需求动态选择 Skills |
| 并行兼容 | 支持多 Skills 同时可用和并行执行 |

详见：[01-skill-registry-middleware.md](01-skill-registry-middleware.md)

---

### 需求 2：标准化技能工厂

**目标**：提升通用能力复用率

| 子需求 | 说明 |
|--------|------|
| SKILL.md 标准化 | 遵循 write-a-skill 规范（<100行，拆分原则） |
| 标准化指令集 | LLM 指令格式 + TOOLS 调用接口标准化 |
| 生命周期管理 | 注册发现 + 加载卸载 + 监控诊断 |

详见：[02-skill-factory.md](02-skill-factory.md)

---

### 需求 3：上下文优化策略

**目标**：降低单次任务的 Token 消耗

| 子需求 | 说明 |
|--------|------|
| 目录注入 | 只注入 SKILL.md 的元数据（TOC） |
| 按需加载 | 基于任务类型/用户查询/LLM 调用触发 |
| 两阶段加载 | 初始化注入目录，运行时加载完整内容 |

详见：[03-context-optimization.md](03-context-optimization.md)

---

### 需求 4：Agent 可观测性建设

**目标**：建立技能调试与链路追踪机制

| 子需求 | 说明 |
|--------|------|
| 链路追踪 | 跨 Skill + Workflow 层追踪 |
| 异常诊断 | P0(可用性/执行/数据) + P1(依赖) |
| 幻觉检测 | 引用源验证 + LLM 自我评估 |
| 污染检测 | 状态快照对比 + 写入白名单 |

详见：[04-observability.md](04-observability.md)

## 技术架构 (Technical Architecture)

### 架构分层

```
┌─────────────────────────────────────────────────────────────────┐
│                    Claude Code CLI                               │
│                   /grill-me  /grill-you                          │
└─────────────────────────┬───────────────────────────────────────┘
                          │
┌─────────────────────────▼───────────────────────────────────────┐
│                      Middleware 调度层                            │
├───────────────────────────────────────────────────────────────────┤
│  请求拦截 │ 上下文管理 │ 技能路由 │ 可观测性收集                  │
└─────────────────────────┬───────────────────────────────────────┘
                          │
        ┌─────────────────▼─────────────────┐
        │         Skill Registry              │
        │  (扫描 SKILL.md，注册技能)          │
        └─────────────────┬─────────────────┘
                          │
        ┌─────────────────▼───────────────────────────────────────┐
        │                  Skills (按需加载)                      │
        │  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐     │
        │  │  grill-me   │  │  grill-you  │  │ advance-t   │     │
        │  │  SKILL.md   │  │  SKILL.md   │  │  SKILL.md   │     │
        │  │  REFERENCE  │  │  REFERENCE  │  │  REFERENCE  │     │
        │  │  scripts/   │  │  scripts/   │  │  scripts/   │     │
        │  └─────────────┘  └─────────────┘  └─────────────┘     │
        └───────────────────────────────────────────────────────┘
```

### 数据流

```
用户请求
    │
    ▼
Middleware (拦截)
    │
    ├─► 识别任务类型
    │
    ├─► 查询 Skill Registry
    │
    ├─► 注入目录元数据
    │
    ├─► 路由到目标 Skill
    │
    ▼
Skill 执行 (按需加载完整内容)
    │
    ├─► 收集可观测性数据
    │
    ▼
返回结果
```

## 成功标准 (Success Criteria)

### 功能验证

- [ ] Skill Registry 可自动发现并注册所有 Skills
- [ ] Middleware 可正确路由 Skill 调用
- [ ] 上下文优化后 Token 消耗降低 ≥ 30%
- [ ] 可观测性数据完整收集链路追踪

- [ ] 测试覆盖率 ≥ 80%
- [ ] 所有 SKILL.md 符合 write-a-skill 规范
- [ ] 可观测性覆盖 P0/P1 异常类型
- [ ] 幻觉检测准确率 ≥ 70%
